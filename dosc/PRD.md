# 暖萌小账本 PRD

## 1. 文档信息

- 产品名称：暖萌小账本
- 项目名称：my-time-ledger
- 文档类型：产品需求文档
- 当前版本：v0.1
- 生成日期：2026-06-17
- 项目形态：移动端优先的 React 单页记账应用原型

## 2. 项目背景

当前项目是一个使用 Vite、React、TypeScript、Tailwind CSS 构建的前端应用，已完成移动端记账产品的静态界面原型。应用通过底部导航切换四个核心页面：首页、目标、记账、我的。

现有实现主要使用本地静态模拟数据，尚未包含真实保存、数据持久化、账单筛选、用户登录、后端接口或测试体系。下一阶段应围绕“让用户可以真实记录、查看和管理个人收支”推进 MVP 产品化。

## 3. 产品定位

暖萌小账本是一款轻量、情绪友好、移动端优先的个人生活记账工具，帮助用户用低压力的方式记录日常收支、查看预算状态、跟踪储蓄目标，并形成持续记账习惯。

## 4. 目标用户

- 想要开始记账但不喜欢复杂财务工具的新手用户。
- 希望快速记录日常消费、收入和储蓄计划的年轻个人用户。
- 偏好可爱、轻松、正向反馈界面的移动端用户。
- 需要用预算目标约束餐饮、购物、旅行、学习等生活支出的用户。

## 5. 用户痛点

- 传统记账工具字段多、流程重，用户容易中途放弃。
- 只记录账单但缺少即时预算反馈，用户难以判断还能花多少。
- 存钱目标和日常消费分离，用户难以感知每笔支出对目标的影响。
- 静态展示无法支持真实使用，账单、目标、个人统计都需要可新增、可计算、可持久化。

## 6. 产品目标

- 让用户在 10 秒内完成一笔账单记录。
- 让首页能基于真实账单展示本月收入、支出、剩余预算和今日账单。
- 让用户能创建并查看储蓄或预算目标进度。
- 让个人中心展示真实累计数据和基础偏好入口。
- 在无后端的前提下，优先通过浏览器本地存储完成 MVP 闭环。

## 7. 非目标

- v0.1 不实现银行账户自动同步。
- v0.1 不实现多人实时协作账本。
- v0.1 不实现复杂投资、资产组合、贷款和信用卡账单管理。
- v0.1 不强制接入后端、登录注册或云同步。
- v0.1 不处理付款、订阅、收费或金融交易功能。

## 8. 当前功能盘点

### 8.1 首页

现有能力：

- 展示应用品牌“暖萌小账本”。
- 展示提醒按钮。
- 展示“可安心使用”金额。
- 展示本月支出、本月收入、剩余预算。
- 展示今日账单列表。
- 提供“查看全部”按钮入口，但尚未实现跳转或列表页。

当前限制：

- 所有金额和账单来自静态数据。
- 提醒按钮无行为。
- “查看全部”无行为。
- 统计数据不会随新增账单变化。

### 8.2 记账页

现有能力：

- 输入金额。
- 选择账单分类：餐饮、交通、购物、生活、收入、储蓄。
- 展示支付账户、日期、票据照片等选项。
- 提供保存账单按钮。

当前限制：

- 分类选择只有默认样式，缺少真实状态管理。
- 保存按钮无提交逻辑。
- 支付账户、日期、票据照片入口无交互。
- 没有输入校验、错误提示和保存成功反馈。

### 8.3 目标页

现有能力：

- 展示本周小目标。
- 展示多个目标卡片。
- 目标卡片包含标题、说明、当前金额、总金额和进度条。

当前限制：

- 目标来自静态数据。
- 无新增、编辑、删除目标能力。
- 无目标完成状态或预算提醒。

### 8.4 我的页

现有能力：

- 展示用户昵称、累计记录笔数。
- 展示总资产、本月结余。
- 展示账本成就、家庭共享、主题装扮、偏好设置入口。

当前限制：

- 所有指标为静态展示。
- 菜单项无实际页面或操作。
- 家庭共享和同步状态只是占位能力。

## 9. MVP 需求

### REQ-001：本地账单数据模型

系统应定义可用于真实记账的数据结构，至少包含：

- 账单 ID
- 类型：支出、收入、储蓄
- 金额
- 分类
- 标题或备注
- 支付账户
- 日期时间
- 创建时间
- 可选票据图片引用

验收标准：

- 每条账单都具备唯一 ID。
- 金额必须大于 0。
- 支出在统计中计入支出，收入计入收入，储蓄可单独归类并影响目标或预算。
- 数据结构可被首页、账单列表和统计模块复用。

### REQ-002：账单新增流程

用户应能在记账页完成一笔账单新增。

功能要求：

- 用户可输入金额。
- 用户可选择分类。
- 用户可选择账单类型，默认按分类推断。
- 用户可填写备注，缺省时使用分类作为标题。
- 用户可选择日期，默认今天。
- 点击保存时进行输入校验。
- 保存成功后清空表单，并给出成功反馈。
- 保存后的账单立即影响首页统计和今日账单。

验收标准：

- 金额为空、非数字或小于等于 0 时，保存失败并显示提示。
- 保存成功后，新账单出现在今日账单列表顶部。
- 不刷新页面时，首页统计随数据变化。
- 刷新页面后，账单仍然存在。

### REQ-003：首页真实统计

首页应基于真实账单数据计算并展示核心指标。

功能要求：

- 本月支出：当前自然月支出合计。
- 本月收入：当前自然月收入合计。
- 剩余预算：月预算减去本月支出。
- 可安心使用：剩余预算或可用余额的友好展示。
- 今日账单：当天账单，按时间倒序展示。

验收标准：

- 新增收入账单后，本月收入增加。
- 新增支出账单后，本月支出增加、剩余预算减少。
- 今日账单只显示当天数据。
- 没有今日账单时展示空状态。

### REQ-004：账单列表入口

用户点击首页“查看全部”后，应能查看完整账单列表。

功能要求：

- 支持按月份展示账单。
- 支持按收入、支出、储蓄筛选。
- 支持按分类筛选。
- 支持空状态。

验收标准：

- “查看全部”入口可访问账单列表。
- 列表按日期倒序排列。
- 筛选后统计和列表内容一致。

### REQ-005：目标管理

目标页应支持用户管理预算或储蓄目标。

功能要求：

- 展示目标列表和进度。
- 支持新增目标，包含标题、说明、当前金额、目标金额、类型和颜色主题。
- 支持编辑目标当前金额。
- 支持目标完成状态展示。
- 支持删除目标。

验收标准：

- 当前金额不能小于 0。
- 目标金额必须大于 0。
- 当前金额超过目标金额时，进度按 100% 展示，并显示完成状态。
- 刷新页面后目标数据仍然存在。

### REQ-006：个人中心真实数据

我的页应展示基于用户数据计算的基础指标。

功能要求：

- 累计记录笔数来自账单数量。
- 总资产可由账户余额或收入支出差额推导。
- 本月结余等于本月收入减本月支出。
- 主题装扮和偏好设置作为后续入口保留。

验收标准：

- 新增账单后累计笔数增加。
- 新增收入或支出后，本月结余实时变化。
- 未配置账户资产时，总资产展示为估算值或空状态，不展示误导性固定值。

### REQ-007：本地持久化

MVP 阶段应使用浏览器本地存储保存账单、目标和基础偏好。

功能要求：

- 使用 localStorage 或 IndexedDB。
- 初始化时加载本地数据。
- 本地无数据时使用空状态或示例数据引导。
- 写入失败时提示用户。

验收标准：

- 刷新页面后数据不丢失。
- 清除浏览器数据后应用能恢复到初始状态。
- 本地存储解析失败时不会导致页面白屏。

### REQ-008：移动端体验和可访问性

应用应保持移动端优先体验，并兼顾基础可访问性。

功能要求：

- 底部导航在主要页面可见。
- 输入框、按钮和筛选控件适合触控。
- 所有图标按钮具备 aria-label 或 title。
- 表单错误提示可被用户明确感知。
- 关键文本在 320px 宽度下不溢出。

验收标准：

- 320px、375px、430px 宽度下页面可正常操作。
- 键盘聚焦状态清晰。
- 表单提交错误不会只依赖颜色表达。

## 11. 页面与交互设计要求

### 首页

- 顶部保留品牌和提醒入口。
- 主卡片展示可安心使用金额。
- 三个统计项展示本月支出、本月收入、剩余预算。
- 今日账单为空时展示轻量空状态和“去记一笔”入口。
- “查看全部”进入账单列表。

### 记账页

- 金额输入应是页面主操作焦点。
- 分类选择应有明确选中态。
- 保存按钮在输入无效时可置灰或点击后提示。
- 保存成功后显示 toast 或 inline 成功反馈。
- 日期选择和账户选择可先用简化弹层或原生控件实现。

### 目标页

- 目标列表保持卡片式展示。
- 目标进度按 current / total 计算。
- 新增目标入口应固定在页面顶部或底部主按钮。
- 完成目标应有清晰但不过度打扰的状态提示。

### 我的页

- 个人数据应优先展示真实统计。
- 暂未实现的入口可保留，但应避免展示“已同步”等误导性状态。
- 设置页可作为后续版本实现。

## 15. 风险与假设

- 当前项目无测试框架，新增测试前需要确定使用 Vitest、Testing Library 或其他工具。
- 当前无路由库，如果新增账单列表页，需要选择继续使用 tab state 还是引入 React Router。
- 本地存储适合 MVP，但不适合多设备同步。
- 票据照片如果真实上传会涉及文件存储、隐私和容量问题，MVP 建议先作为占位或本地预览。
- 家庭共享、同步、提醒等功能涉及权限、通知或后端能力，建议后续版本再实现。

## 16. 验收清单

- 用户可以新增一笔合法账单。
- 用户无法保存非法金额。
- 新增账单后首页统计立即更新。
- 新增账单后今日账单立即展示。
- 刷新页面后账单数据不丢失。
- 首页无账单时有空状态。
- 用户可以查看目标进度。
- 用户可以新增或更新目标进度。
- 个人中心累计记录笔数来自真实账单数量。
- 移动端 320px 宽度无明显文字溢出和控件重叠。

## 17. 开放问题

- 月预算的默认值应由系统给出，还是要求用户首次设置？
- 储蓄类账单是否应自动关联某个目标？
- 是否需要账单删除和编辑能力进入 v0.1？
- “提醒”功能是否只做入口，还是要接入浏览器通知？
- 账单列表页应作为第五个页面，还是从首页入口临时覆盖当前内容？
